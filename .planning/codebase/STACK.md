# Stack

DeepBook v3 is a decentralized order book on the Sui blockchain with an off-chain indexer and REST API server.

## Languages

| Language | Version | Where used |
|----------|---------|-----------|
| Move | Sui Move (2024 edition) | `packages/` — all on-chain smart contracts |
| Rust | Edition 2021 | `crates/` — indexer, server, schema, bench |
| TypeScript | ^5.6.3 | `scripts/` — transaction scripts, deployment tooling |

## On-Chain (Move)

**Packages:**

| Package | Purpose |
|---------|---------|
| `packages/deepbook` | Core order book: pool, book, orders, balance manager, vault, governance, EWMA, DEEP price |
| `packages/deepbook_margin` | Margin trading extension: margin pools, oracle integration, liquidation |
| `packages/predict` | Prediction markets: vaults, oracle, PLPs, range/market keys |
| `packages/token` | DEEP governance token (TreasuryCap, ProtectedTreasury) |
| `packages/dbtc` | Wrapped BTC coin |
| `packages/dusdc` | Wrapped USDC coin |
| `packages/margin_liquidation` | Liquidation vault logic |

**Move dependencies (from `Move.toml`):**
- `sui-framework` (from Sui git, testnet branch)
- Inter-package deps: `deepbook_margin` depends on `deepbook`; `predict` depends on `deepbook`; etc.

## TypeScript / Scripts

**Runtime:** Node.js with `tsx` for direct TS execution

**Key dependencies:**
- `@mysten/sui` (implied by scripts) — Sui TypeScript SDK
- `prettier` ^3.3.3 + `@mysten/prettier-plugin-move` ^0.3.5 — code formatting
- `eslint` stack (typescript-eslint, import sorting, header checking)
- `ts-node` / `tsx` ^4.19.2 — script execution

**Scripts directory layout:**
- `scripts/transactions/` — ~40+ individual transaction scripts (createPool, enableVersion, etc.)
- `scripts/tx/` — lower-level tx builders
- `scripts/utils/` — shared utilities
- `scripts/config/` — environment configuration

## Database

**PostgreSQL** (version not pinned in codebase)
- Accessed via `diesel-async` with `bb8` connection pool
- Schema auto-generated by `diesel print-schema` in `crates/schema/src/schema.rs`
- 50+ tables (one per indexed event type) + materialized views (`net_deposits_hourly`, `ohclv_1m`, `ohclv_1d`)
- Stored function: `get_ohclv()` for OHLCV candle aggregation
- Migrations: `diesel_migrations` (path not verified, assumed standard `migrations/`)
