<objective>
Split `crates/server/src/server.rs` — a 2,857-line monolith — into domain sub-modules under
`crates/server/src/routes/`. This is the foundational change for Phase 2: all other plans
(SCALE-02, SCALE-03, PERF-05, DX-03) work on the post-split module structure. After this plan,
`server.rs` retains only infrastructure (AppState, path constants, run_server, make_router) while
handler functions live in their domain modules.

Purpose: The monolith makes it hard to find, test, and modify individual endpoint groups. The
split follows the existing pattern in `crates/server/src/admin/routes.rs`.
</objective>

Existing split pattern to follow exactly:
- crates/server/src/admin/routes.rs — sub-module that returns Router<Arc<AppState>> (unbound)
- Key rule: NO .with_state() inside sub-module routers. Single .with_state() in make_router after merge.
- AppState stays in server.rs (used by run_server). Routes import it via `use crate::server::AppState;`
- No import cycle: routes import from server (AppState), server imports route constructors from routes.
- ParameterUtil trait (server.rs lines 2071-2109) moves to routes/mod.rs so all sub-modules can use it.
- Path constants (server.rs lines 66-124) stay in server.rs, imported in sub-modules via `use crate::server::*;` or named imports.
