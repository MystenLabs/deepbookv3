<objective>
Add `moka` in-process caching to the two hot endpoints: `get_pools` (pool metadata, TTL 60s) and
`ticker` (per-pool ticker, TTL 10s). Pool metadata is fetched on every request to get_pools,
ticker, summary, historical_volume, all_historical_volume, and trades — caching it eliminates
redundant DB reads on repeat requests.

Purpose: DB reads for stable data (pool metadata changes rarely) are wasteful under load.
`moka` is the decided library (STATE.md); no Redis, no manual Arc<Mutex<HashMap>>.
</objective>

moka 0.12 API (verified from crates.io/docs.rs):
```rust
use moka::future::Cache;
use std::time::Duration;

let cache: Cache<K, V> = Cache::builder()
    .max_capacity(n)
    .time_to_live(Duration::from_secs(ttl))
    .build();

// read
if let Some(value) = cache.get(&key).await {
    return Ok(Json(value));
}
// miss — compute and insert
let value = expensive_call().await?;
cache.insert(key, value.clone()).await;
Ok(Json(value))
```

Pitfall: moka requires V: Clone + Send + Sync + 'static.
- Vec<Pools>: requires Pools to be Clone. Check existing derives on Pools struct in reader.rs.
  If Pools is not Clone, add #[derive(Clone)] to it.
- serde_json::Value: is Clone + Send + Sync + 'static ✓

Pitfall: pools_cache key is () (unit) since there's only one global pool list.
Pitfall: AppState must remain Clone-compatible. moka::future::Cache is Clone (internally Arc-wrapped) ✓.
Pitfall: Do not use #[derive(Clone)] on AppState if it has non-Clone fields — verify first.

Cache initialization: AppState is constructed in run_server or just before make_router is called.
Find that location in server.rs and add the cache fields to the AppState struct literal.
</context>
