Axum 0.7 FromRequestParts pattern:
```rust
use axum::{
    async_trait,
    extract::{FromRequestParts, Path},
    http::request::Parts,
};

pub struct ValidatedWalletAddress(pub String);

#[async_trait]
impl<S> FromRequestParts<S> for ValidatedWalletAddress
where
    S: Send + Sync,
{
    type Rejection = DeepBookError;

    async fn from_request_parts(parts: &mut Parts, state: &S) -> Result<Self, Self::Rejection> {
        let Path(wallet_address) = Path::<String>::from_request_parts(parts, state)
            .await
            .map_err(|_| DeepBookError::bad_request("Missing wallet address"))?;
        if !wallet_address.starts_with("0x")
            || wallet_address.len() != 66
            || !wallet_address[2..].chars().all(|c| c.is_ascii_hexdigit())
        {
            return Err(DeepBookError::bad_request(
                "Invalid wallet address: expected 0x-prefixed 64-character hex string",
            ));
        }
        Ok(ValidatedWalletAddress(wallet_address))
    }
}
```

| Boundary | Description |
|----------|-------------|
| HTTP request → Axum extractor | Wallet address and limit are validated at the HTTP boundary |
| Axum extractor → reader.rs | reader.rs receives a pre-validated wallet address; no double-validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05-01 | Spoofing | Wallet address injection | mitigate | ValidatedWalletAddress enforces 0x-prefix + 64-char hex — only valid Sui addresses pass; rejects path traversal attempts |
| T-02-05-02 | Denial of Service | Oversized limit parameter | mitigate | PaginationParams.validated_limit() clamps to max 10000 — prevents runaway DB queries |

</threat_model>
