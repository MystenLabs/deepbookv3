<objective>
Add `asset_normalized TEXT NOT NULL DEFAULT ''` to the `balances` table with a B-tree index, populate it via a separate backfill script, update `schema.rs`, and rewrite the portfolio collateral sub-query to join on `asset_normalized` instead of using a leading-wildcard LIKE.

Purpose: The portfolio collateral query joins `balances` to `asset_meta` using `b.asset LIKE '%' || SUBSTRING(am.asset_id FROM 3) || '%'`. A leading `%` wildcard forces PostgreSQL to do a sequential scan on `balances.asset`. Replacing this with an equality join on a normalized column enables the B-tree index and eliminates the seq scan.

Output: One migration (column + index), one standalone backfill SQL script, updated schema.rs, updated reader.rs collateral sub-query.

IMPORTANT — backfill is NOT in the migration (PITFALL P11 from PITFALLS.md): the balances table is too large to backfill inside a migration transaction without risking timeout or lock issues. The migration only adds the column and index. The backfill script runs separately after migration.
</objective>

Normalization rule (must be consistent between backfill and query):
- Strip the leading `0x` prefix if present
- Lowercase the result
- Example: `0xABCD1234...` → `abcd1234...`
- PostgreSQL expression: `LOWER(REGEXP_REPLACE(asset, '^0[xX]', ''))`

After migration+backfill, the collateral JOIN changes from:
  `LEFT JOIN asset_meta am ON b.asset LIKE '%' || SUBSTRING(am.asset_id FROM 3) || '%'`
to:
  `LEFT JOIN asset_meta am ON b.asset_normalized = LOWER(REGEXP_REPLACE(am.asset_id, '^0[xX]', ''))`

The `asset_meta` CTE already strips the `0x` prefix via `SUBSTRING(am.asset_id FROM 3)` — the new
join replaces that LIKE with an equality comparison using the same normalized form.

Note: The `asset_normalized` column will be empty string for any rows inserted AFTER the migration
but BEFORE the backfill. The backfill script must be run promptly after migration. New rows inserted
by the indexer after Phase 2 (SCALE-03) will be written correctly via `normalize_asset_id()`.
For this phase, the column serves the backfill+query fix; indexer write-path normalization is SCALE-03.
</context>
